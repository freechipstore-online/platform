ISA Overview

RISC-V (pronounced "risk-five") is an open, free instruction set architecture (ISA) that originated at UC Berkeley in 2010. Created by Krste Asanovic, Andrew Waterman, and Yunsup Lee, it was the fifth RISC ISA designed at Berkeley (after RISC-I, RISC-II, SOAR, and SPUR). Unlike ARM or x86, RISC-V is not owned by any company — it is governed by RISC-V International, a non-profit foundation with 4,000+ member organizations.

Why RISC-V Matters

  • Open and free — no license fees, no royalties, no permission needed. Anyone can design, manufacture, and sell RISC-V processors. This eliminates the vendor lock-in of ARM (royalties per chip) and x86 (Intel/AMD duopoly).
  • Clean-slate design — learned from 40 years of ISA mistakes. No backward-compatibility baggage (unlike x86 carrying 8086-era instructions). Modular: small base ISA plus optional standard extensions.
  • Modular and extensible — the base ISA is intentionally minimal (47 instructions for RV32I). Extensions add functionality: M (multiply/divide), A (atomics), F/D (floating-point), C (compressed), V (vector), and custom extensions for domain-specific accelerators.
  • Growing ecosystem — Linux, FreeBSD, Zephyr RTOS, FreeRTOS. GCC, LLVM, Rust compiler. QEMU, Spike simulator. Android support. Billions of RISC-V cores shipped (mostly in microcontrollers).

RISC-V vs ARM vs x86

  • x86-64 — CISC, variable-length instructions (1-15 bytes), 400+ instructions, enormous backward compatibility surface. Dominant in desktops/servers. Intel and AMD only.
  • ARM (AArch64) — RISC-ish, fixed 32-bit instructions, large ISA (~1,000 instructions including extensions). Dominant in mobile/embedded. Requires ARM license (~$1-5M + per-chip royalty).
  • RISC-V — pure RISC, fixed 32-bit instructions (+ 16-bit compressed), minimal base ISA. Open and free. Growing from embedded into application processors. No single vendor controls it.

Base Integer Instructions (RV32I)

RV32I is the base 32-bit integer instruction set. It defines 47 instructions, 32 integer registers (x0-x31, each 32 bits wide), and a 32-bit program counter. Register x0 is hardwired to zero — reads always return 0, writes are discarded. This simplifies instruction encoding and eliminates the need for a separate "move" instruction (use add rd, rs, x0).

Instruction Categories

  • Arithmetic — ADD, SUB, ADDI (add immediate). No flags register — comparisons are done with branch instructions or SLT (Set Less Than).
  • Logical — AND, OR, XOR, ANDI, ORI, XORI.
  • Shift — SLL (shift left logical), SRL (shift right logical), SRA (shift right arithmetic). Immediate variants: SLLI, SRLI, SRAI.
  • Compare — SLT (set less than, signed), SLTU (unsigned), SLTI, SLTIU. Sets destination register to 1 if condition holds, 0 otherwise.
  • Load/Store — LW (load word), LH/LHU (load halfword signed/unsigned), LB/LBU (load byte). SW, SH, SB for stores. Load/store architecture: only these instructions access memory.
  • Branch — BEQ (branch if equal), BNE (not equal), BLT/BLTU (less than signed/unsigned), BGE/BGEU (greater or equal). PC-relative, 12-bit signed offset (range: -4096 to +4094 bytes).
  • Jump — JAL (jump and link: saves return address to rd), JALR (jump and link register: indirect jump). Used for function calls and returns.
  • Upper immediate — LUI (load upper immediate: loads a 20-bit constant into the upper 20 bits of rd), AUIPC (add upper immediate to PC). Together with ADDI, these construct arbitrary 32-bit constants and addresses.
  • System — ECALL (system call), EBREAK (debugger breakpoint), FENCE (memory ordering).

RV64I

The 64-bit variant. Registers are 64 bits wide. Adds word-sized variants: ADDW, SUBW, SLLW, SRLW, SRAW (operate on the lower 32 bits and sign-extend the result), plus LWU (load word unsigned) and LD/SD (load/store doubleword). RV64I is the target for Linux-capable application processors.

Standard Extensions

MMultiply/DivideMUL, MULH, MULHU, MULHSU, DIV, DIVU, REM, REMU. Hardware integer multiplication and division. Essential for general-purpose computing.
AAtomicLR (Load-Reserved), SC (Store-Conditional), and AMO (Atomic Memory Operations: AMOSWAP, AMOADD, AMOAND, AMOOR, AMOXOR, AMOMAX, AMOMIN). For multi-core synchronization, mutexes, lock-free data structures.

DDouble-Precision FloatExtends F to 64-bit doubles. Same operations with .D suffix. f registers are 64 bits wide. Requires F extension.

The "G" Shorthand

RV32G = RV32IMAFDZicsr_Zifencei — the "general-purpose" ISA combining the base integer ISA with multiply, atomics, single and double float, CSR access, and instruction fence. This is the standard target for Linux-capable processors. RV64G is the 64-bit equivalent.

Custom Extensions

RISC-V explicitly reserves opcode space for custom extensions (four major opcodes reserved). Companies can add domain-specific instructions without conflicting with standard extensions. Examples: Qualcomm Hexagon-style VLIW extensions, AI accelerator instructions, cryptographic instructions. Custom extensions are a key advantage over ARM and x86, where adding instructions requires vendor approval.

Instruction Formats

RISC-V uses 6 instruction formats, all 32 bits wide. The opcode field (bits 6:0) is always in the same position. Source registers (rs1, rs2) and destination register (rd) are always in the same bit positions when present. This regularity simplifies decode hardware.

  • R-type — register-register operations: [funct7 | rs2 | rs1 | funct3 | rd | opcode]. Used for ADD, SUB, AND, OR, SLL, SLT, etc.
  • I-type — immediate operations and loads: [imm[11:0] | rs1 | funct3 | rd | opcode]. 12-bit signed immediate. Used for ADDI, LW, JALR, etc.
  • S-type — stores: [imm[11:5] | rs2 | rs1 | funct3 | imm[4:0] | opcode]. Immediate split across two fields (no rd needed for stores).
  • B-type — branches: [imm[12|10:5] | rs2 | rs1 | funct3 | imm[4:1|11] | opcode]. 12-bit signed offset, always even (bit 0 implicitly 0). Allows +/- 4 KiB range.
  • U-type — upper immediate: [imm[31:12] | rd | opcode]. 20-bit immediate placed in upper bits. Used for LUI, AUIPC.
  • J-type — jumps: [imm[20|10:1|11|19:12] | rd | opcode]. 20-bit signed offset. JAL can jump +/- 1 MiB.

Design Rationale

The consistent register field positions (rs1 at bits 19:15, rs2 at bits 24:20, rd at bits 11:7) allow the register file to be read before the instruction is fully decoded. The immediate encoding places the sign bit always at bit 31 for fast sign-extension. These seemingly minor choices significantly simplify high-performance pipeline implementations.

Privileged Architecture

The RISC-V privileged specification defines the machine modes, interrupt handling, virtual memory, and system-level features needed to run operating systems.

Privilege Levels

  • Machine mode (M) — highest privilege. Always present. Has full access to all hardware and CSRs. Firmware, bootloader, and bare-metal code run here. Can trap and emulate missing features.
  • Supervisor mode (S) — optional. OS kernel runs here. Can access its own CSRs and manage virtual memory, but cannot directly access M-mode CSRs. Traps to M-mode for firmware services (SBI — Supervisor Binary Interface).
  • User mode (U) — optional. Application code runs here. Lowest privilege. Memory access controlled by page tables set up by S-mode. System calls trap to S-mode.
  • Hypervisor extension (H) — adds VS (Virtual Supervisor) and VU (Virtual User) modes for hardware-assisted virtualization. A hypervisor in HS-mode manages multiple guest OSes, each running in VS/VU-mode.

CSRs (Control and Status Registers)

  • mstatus/sstatus — global interrupt enable, privilege mode tracking, FP state (dirty/clean)
  • mtvec/stvec — trap vector base address (where execution jumps on interrupt/exception)
  • mepc/sepc — exception program counter (return address for MRET/SRET)
  • mcause/scause — cause of the trap (interrupt bit + exception code)
  • mie/sie, mip/sip — interrupt enable and pending bits (timer, software, external interrupts)
  • satp — supervisor address translation and protection. Points to the root page table. Controls virtual memory mode (Bare, Sv32, Sv39, Sv48, Sv57).
  • Performance counters — mcycle (clock cycles), minstret (instructions retired), mhpmcounter3-31 (configurable hardware performance counters).

Virtual Memory

  • Sv32 — 32-bit virtual addresses, 2-level page table, 4 KiB pages, 4 MiB megapages. For RV32.
  • Sv39 — 39-bit virtual addresses (512 GiB address space), 3-level page table, 4 KiB / 2 MiB / 1 GiB pages. The standard for RV64 Linux.
  • Sv48 — 48-bit virtual addresses (256 TiB), 4-level page table. For larger address spaces.
  • Sv57 — 57-bit virtual addresses (128 PiB), 5-level page table. For future large-memory systems.
  • TLB (Translation Lookaside Buffer) caches page table entries. SFENCE.VMA instruction flushes TLB entries (with optional ASID and address filtering).

Ecosystem

Software

  • Linux — full RISC-V port in mainline kernel since v5.17 (2022). Runs on QEMU, SiFive boards, StarFive VisionFive 2, and Milk-V Pioneer. Debian, Ubuntu, Fedora, Arch Linux all have RISC-V ports.
  • Toolchains — GCC (full support since GCC 7), LLVM/Clang (since LLVM 9), Rust (tier 2 support for riscv64gc-unknown-linux-gnu). Go, Java (OpenJDK), Python, Node.js all run on RISC-V.
  • RTOS — FreeRTOS, Zephyr, RT-Thread, NuttX. Essential for embedded RISC-V microcontrollers.
  • Simulators — Spike (reference ISA simulator from Berkeley), QEMU (user-mode and full-system emulation), Renode (multi-node SoC simulation), and gem5 (cycle-accurate microarchitecture simulation).
  • Android — AOSP RISC-V port in progress. Google announced official RISC-V support. Android 15 builds run on QEMU. Commercial Android RISC-V phones expected by 2026-2027.

SBI (Supervisor Binary Interface)

The SBI is the firmware interface between S-mode (OS kernel) and M-mode (firmware). It provides standardized services: console I/O, timer, IPI (inter-processor interrupt), remote fence, HSM (hart state management), and PMU (performance monitoring). OpenSBI is the reference open-source SBI implementation. Analogous to BIOS/UEFI services in the x86 world.

Debug Specification

RISC-V defines a standard debug interface: debug module (DM), debug transport module (DTM), and trigger module. Supports external debug (via JTAG) and self-hosted debug (ebreak). GDB connects via OpenOCD. Hardware breakpoints (up to implementation-defined count) and single-stepping are specified.

Implementations

Open-Source Cores

Rocket

In-order, 5-stage pipeline. Written in Chisel (Scala). The reference RISC-V implementation from UC Berkeley. Configurable, well-tested, and silicon-proven.

UC Berkeley | Chisel | BSD

BOOM

Berkeley Out-of-Order Machine. Superscalar, out-of-order, speculative. Written in Chisel. Research-grade implementation competitive with commercial cores.

UC Berkeley | Chisel | BSD

CVA6 (Ariane)

6-stage, single-issue, in-order pipeline. SystemVerilog. Linux-capable. Maintained by OpenHW Group. Used in academic and industrial research.

ETH Zurich | SystemVerilog | Apache 2.0

VexRiscv

Highly configurable RISC-V core. Written in SpinalHDL. From tiny (RV32E, 400 LUTs) to Linux-capable (RV32IMA, MMU). FPGA-optimized.

SpinalHDL | MIT License

PicoRV32

Tiny RISC-V core (RV32IMC). ~2000 lines of Verilog. Optimized for area (750-2000 LUTs). No pipeline — multicycle. Great for learning and tiny FPGA projects.

Clifford Wolf | Verilog | ISC

NEORV32

Full-featured RISC-V processor in VHDL. RV32/RV64, M/S/U modes, MMU, FPU, peripherals included. Targets any FPGA. Excellent documentation.

VHDL | BSD | FPGA-optimized

Commercial Implementations

  • SiFive — the first commercial RISC-V company (founded by RISC-V creators). U74 (Linux-capable, quad-core, in JH7110 SoC), P870 (high-performance, out-of-order, targeting mobile). SiFive licenses IP cores to chip companies.
  • Alibaba T-Head (Xuantie) — C910 (high-performance, 12-stage, out-of-order, 2.5 GHz, open-sourced). C906 (Linux-capable, in-order). Used in Alibaba cloud servers and IoT. Largest volume RISC-V application processor deployment.
  • Ventana Micro — Veyron V1 (data-center-class, 64-core, 192-core chiplet, targeting cloud/HPC). Built on TSMC 5nm. Competitive with ARM Neoverse in server workloads.
  • Tenstorrent — Ascalon (high-performance RISC-V core, out-of-order, designed by Jim Keller). Targets AI/ML accelerators and general-purpose computing.
  • WCH — CH32V series. Low-cost RISC-V microcontrollers ($0.10-$2). Huge volume in Chinese consumer electronics. CH32V003 is a 32-bit RISC-V MCU at $0.10.
  • Espressif — ESP32-C3 and ESP32-C6 use a single-core RISC-V processor with WiFi/BLE. Popular in IoT/hobbyist market. Millions shipped.
  • Qualcomm — announced RISC-V adoption for future wearable and IoT chips. Founding member of RISC-V International.

RISC-V in Numbers

  • Over 10 billion RISC-V cores shipped as of 2024 (mostly in microcontrollers and embedded SoCs)
  • 4,000+ member organizations in RISC-V International
  • 50+ RISC-V-based products available for purchase (dev boards, SBCs, microcontrollers)
  • RISC-V laptop (ROMA by DeepComputing) shipping with Debian Linux
